1. Objectif

Ce document définit les règles d’utilisation, d’organisation et de rédaction des prompts utilisés dans le projet.

Il garantit des interactions cohérentes avec les agents IA et facilite leur réutilisation.

⸻

2. Principes

Un prompt doit être :

* clair ;
* précis ;
* contextualisé ;
* non ambigu ;
* orienté vers un objectif.

Il doit fournir suffisamment d’informations pour permettre à l’agent de produire une réponse pertinente.

⸻

3. Structure recommandée

Un prompt peut contenir les éléments suivants :

* le contexte ;
* l’objectif ;
* les contraintes ;
* les informations disponibles ;
* le résultat attendu.

Tous les éléments ne sont pas obligatoires, mais ils doivent être utilisés lorsque cela améliore la compréhension de la demande.

⸻

4. Bonnes pratiques

Les prompts doivent :

* éviter les ambiguïtés ;
* préciser les hypothèses importantes ;
* indiquer les contraintes connues ;
* définir clairement le résultat attendu.

Lorsque plusieurs étapes sont nécessaires, elles doivent être présentées dans un ordre logique.

⸻

5. Réutilisation

Les prompts fréquemment utilisés doivent être rédigés de manière suffisamment générique pour être adaptés à différents contextes.

Les duplications doivent être évitées.

⸻

6. Évolution

Les prompts peuvent évoluer avec le projet.

Toute amélioration significative doit être documentée et validée avant d’être adoptée comme référence.

⸻

7. Références

Les prompts s’appuient notamment sur :

* SYSTEM.md
* RULES_OF_ENGAGEMENT.md
* AGENT.md
* WORKFLOW.md
* CONTEXT.md

Ils doivent rester cohérents avec les principes définis dans ces documents.
